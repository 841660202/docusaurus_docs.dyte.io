This guide will provide you with step-by-step instructions on starting and stopping livestreaming for your Dyte meetings using simple API calls. These API calls provide you with all the necessary components to start broadcasting a live stream and ensure high-quality playback for your audience.

## Prerequisites

Before getting started with this guide, we recommend that you review the following topics:

- [Getting Started with Dyte](/getting-started) to familiarize yourself with Dyte
- [Introduction to Dyte's Interactive Livestreaming](/guides/livestream/livestream-overview)

## Start a livestream

To start livestreaming a meeting, do the following:

### Step 1: Get authentication token

Get authentication token for your organization using Developer Portal. For more information, see [Create a Dyte Developer Account](/guides/livestream/interactive-livestream/getting-started-with-livestreaming#step-1-create-a-dyte-developer-account).

### Step 2: Create a live stream

You simply need to specify the `meeting ID` of the meeting that you want to live stream. To fetch the meeting ID, you can do the following:

- If you've already created the meeting, use the [Fetch all meetings](/api/?v=v2#/operations/get_all_meetings) API to view all the meetings for your organization.
- If you've not created a meeting yet, use the [Create a meeting](/api/?v=v2#/operations/create_meeting) API to create the meeting that you want to live stream.

These API returns a parameter called `id`, which is your `meeting ID`.

:::info note

With Dyte, you can also specify to begin live streaming the meeting as soon as it starts. To do so, specify
`live_stream_on_start` parameter in the
[Create a meeting API](/api/?v=v2#/operations/create_meeting) when creating a meeting.

:::

#### Request

In the [Start livestreaming a meeting](/api#/paths/meetings-meeting_id--livestreams/post) specify the `meeting ID` of the meeting that you want to live stream as the following:

```
curl --request POST \
  --url https://api.cluster.dyte.in/v2/meetings/{meetingId}/livestreams \
  --header 'Authorization: Basic <token>' \
  --header 'Content-Type: application/json'
```

## Fetch details of an active livestream

Fetching details of an active livestream is also an HTTP request away.

### Request

Specify the `meeting_id` of the meeting for which you want to fetch the active livestream.

Use the [Fetch all meetings](/api/?v=v2#/operations/get_all_meetings) or [Create a meeting](/api/?v=v2#/operations/create_meeting) API to
get the **meeting ID**. The API returns a parameter called `id`, which is your
`meeting ID`.

```bash
curl --request GET \
  --url https://api.cluster.dyte.in/v2/meetings/{meetingId}/active-livestream \
  --header 'Authorization: Basic <token>' \
  --header 'Content-Type: application/json'
```

### Response

```json
{
  "success": true,
  "data": {
    "status": "LIVE",
    "name": null,
    "meeting_id": "188566bc-8770-4ca0-9eb3-24b594ecaf7e",
    "ingest_seconds": 0,
    "viewer_seconds": 0,
    "ingest_server": "rtmps://2ec802dd47b0.global-contribute.live-video.net:443/app/",
    "id": "7088bba8-f522-49a8-b59b-3cd0e946bbb0",
    "stream_key": "sk_ap-south-1_AfnVpeYZjidO_fBpWKody7sDUOEfy9Zc6XAZegkR0Va",
    "playback_url": "https://2ec802dd47b0.ap-south-1.playback.live-video.net/api/video/v1/ap-south-1.944908621410.channel.rVKA2KM5Lunt.m3u8"
  }
}
```

You can also use the [Fetch all livestreams for a meeting](/api#/paths/meetings-meeting_id--livestreams/get) API or [Fetch all livestreams](/api#/operations/fetch_all_livestreams) to get livestream details.

## Stop a livestream

To stop live streaming of a meeting, provide the `meeting_id` of that meeting in the following request.

Use the [Fetch all meetings](/api/?v=v2#/operations/get_all_meetings) or [Create a meeting](/api/?v=v2#/operations/create_meeting) API to
get the **meeting ID**. The API returns a parameter called `id`, which is your
`meeting ID`.

### Request

```bash
curl --request POST \
  --url https://api.cluster.dyte.in/v2/meetings/meetingId/active-livestream/stop \
  --header 'Authorization: Basic <token>' \
  --header 'Content-Type: application/json'
```

### Response

```json
{
  "success": true,
  "data": {
    "message": "livestream stopped"
  }
}
```
