Dyte's live video platform, built upon [WebRTC](https://dyte.io/blog/webrtc/), enables you to seamlessly incorporate real-time communication features into your web and mobile applications. Using Dyte's REST APIs and client-side SDKs, you can effortlessly acquire, transmit, and play media while efficiently managing network connections within your applications. All Dyte live video calls come with in-built noise cancellation (powered by Krisp).

On this page, you will find an overview to help you craft an exceptional user experience with minimal hassle. To kickstart your journey, the [Essentials](#essentials) section will guide you through the setup process and fundamental concepts. In the [Capabilities](#capabilities) section, you'll discover functionalities that you can incrementally add to enhance your live video call experience.

Dyte offers a prebuilt library of UI components that simplifies the integration of live video calls into any platform within minutes. In the [Prebuilt UI](#prebuilt-ui) section, you'll find guides on integrating them into your existing applications. There are also guides that can help you customise the components to align with your brand's identity.

Explore the [Reference](#reference) section to build a better understanding of the core SDK that provides simple APIs for complex media and networking optimizations driving your live video experience. Additionally, you can access references for the REST APIs that power your video calls.

## Supported platforms

Dyte currently supports the following platforms:

- **Mobile**: Flutter, Android (Java/Kotlin), iOS (Objective-C/Swift), React Native
- **Web**: Javascript Core SDK + UI Kit for React JS, Angular, and Web Components for other frameworks
- **Desktop**: Electron
