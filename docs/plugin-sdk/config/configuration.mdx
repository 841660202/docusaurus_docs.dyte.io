When creating a plugin using [Dyte CLI](https://docs.dyte.io/cli), a config file called `dyte-config.json` is auto-generated for you.
This is used to customise the plugin by adding a name, description, tags and much more.

### Generating `dyte-config`

A basic config file is auto-generated for you when you create a plugin using [Dyte CLI](https://docs.dyte.io/cli/plugins) .


:::danger important
It is important to generate this using the [Dyte CLI](https://docs.dyte.io/cli/plugins).
:::

### Properties of `dyte-config`

Here is a list of all properties the config object can have:

| **Property** | **Descriptio**n | **Type** | **Required** |
| --- | --- | --- | --- |
| name | Name of the plugin. | string | true |
| version | | string | true |
| description | A brief about your plugin, this shows in the plugins tab of the meeting. | string | true |
| PluginId | A unique identifier generated by Dyte. | string | true |
| picture | URL for plugin icon. | string | true |
| url | URL where the plugin would be deployed. Auto Generated. | string | false |
| files | An object with keys include and exclude which store a list of relative paths to files. | Object | true |
| tags | List of relevant keywords or topics. | string[] | true |
| staggered | Staggered plugins are enabled only for the user who opens them. | boolean | false |
| author | Name of the creator. | string | false |
| repository | URL for the Github repository. | string | false |
| contentScript | Relative path to the file that uses MediaStreams API. | string | false |


This is what a basic config object looks like:

```tsx
{
    "name": "my-plugin",
    "version": "0.0.X",
    "description": "Add a short description for your plugin here",
    "pluginId": "79936149-758e-4d8d-8366-23deaxxxxxxx",
    "picture": "https://dyte-uploads.s3.ap-south-1.amazonaws.com/dyte.png",
    "url": "https://79936149-758e-4d8d-8366-23deaxxxxxxx-latest.plugins.dyte.app",
    "files": {
        "include": [
            "index.html",
            "**/*"
        ],
        "exclude": [
            "**/*.ts",
            "*.md"
        ]
    },
    "staggered": false,
    "tags": ["#plugin"],
    "author": "",
    "repository": "",
    "contentScript": ""
}
```

### Updating `dyte-config`

You can update the meta-data of your plugin using the following command. This generates a new `dyte-config.json`.
```shell
dyte plugins update
```
