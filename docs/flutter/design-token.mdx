# Design System

A design system ins a collection of customizable methods, guided by clear standards, that can be used to build any number of application with your own personalization touch.

Dyte's UI Kit uses Atomic design as a principle to bring logic and structure to individual screens. Read this blog pos to get a complete overview of how we're using design tokens and atomic design principles to create a multi-branch, multi-device design system.

Let's look at how you can customize your UI with Dyte's Flutter UI Kit.


## Customizing with Design Tokens - Changing themes

Dyte Provided a `DyteDesignToken` class to customize the theme of the UI Kit. You can pass the `DyteDesignToken` object in `DyteUIKitInfo` object to customize the theme of the UI Kit.

`DyteDesignToken` has the following parameters:

- `backgroundColor`: The background color of the UI Kit.
- `textOnBackground`: The text color on the background of the UI Kit.
- `backgroundColorSwatch`: The background color swatch of the UI Kit.
- `brandColor`: The primary color of the UI Kit.

- `textOnBrand`: The text color on the brand color of the UI Kit.
- `brandColorSwatch`: The brand color swatch of the UI Kit.
- `danger`: The danger color of the UI Kit.
- `success`: The success color of the UI Kit.
- `warning`: The warning color of the UI Kit.

:::info
To customize the theme of the uikit, you can either pass color or color swatch. You can't pass both at the same time.

To pass brand color swatch, you need to pass the color swatch with 5 entries with keys 300, 400, 500, 600, 700.

To pass background color swatch, you need to pass the color swatch with 5 entries with keys 600, 700, 800, 900, 1000.

This is done through the `DyteColorSwatch` class.
:::

`DyteColorSwatch` class has two parameters, the first parameter is the default color key and the second parameter is the map of colors.

### Example

```dart
final primarySwatch = DyteColorSwatch(
    500,
    {
      300: Colors.blue.shade300,
      400: Colors.blue.shade400,
      500: Colors.blue.shade500,
      600: Colors.blue.shade600,
      700: Colors.blue.shade700,
    },
  );
```



## Customizing with Configurations - Changing borders.

You can pass the `DyteBorderRadius` and `BorderWidth` enum values in the design token to customize the border radius and border width of the UI Kit.

`borderRadius`: The border radius of the UI Kit. It has the following values.
  - `DyteBorderRadius.sharp`
  - `DyteBorderRadius.rounded`
  - `DyteBorderRadius.extrarounded`
  - `DyteBorderRadius.circular`


`borderWidth`: The border width of the UI Kit.
  - `BorderWidth.none`
  - `BorderWidth.thin`
  - `BorderWidth.fat`


## Example

A demo of how you can pass the `DesignToken` in Flutter UI Kit is shown below:


```dart
    final uikitInfo = DyteUIKitInfo(meetingInfo,
        designToken: DyteDesignToken(
          colorToken: DyteColorToken(
              borderRadius: DyteBorderRadius.circular,
              borderWidth: BorderWidth.fat,
              backgroundColor: Colors.black,
              textOnBrand: Colors.white,
              textOnBackground: Colors.white,
              danger: Colors.red,
              success: Colors.green,
              warning: Colors.yellow,
              brandColorSwatch: DyteColorSwatch(
                500,
                {
                  300: Colors.blue.shade300,
                  400: Colors.blue.shade400,
                  500: Colors.blue.shade500,
                  600: Colors.blue.shade600,
                  700: Colors.blue.shade700,
                },
              )),
        ));
```
