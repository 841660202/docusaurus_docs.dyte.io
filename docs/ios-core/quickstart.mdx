# iOS Core SDK Quickstart

This quickstart shows how to use Dyte's iOS Core SDK to add live video and audio
to your iOS applications.

For getting started quickly, you can use our
[sample code](https://github.com/dyte-io/mobile-samples/tree/main/iOS-core). You
can clone and run a sample application from the iOS Core SDK GitHub repository.

## Step 1: Install the SDK

import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

:::tip iOS minimum version 12.0

We support minimum OS version `12.0` for our iOS SDK.

:::

1. Add the following line to your `Podfile`

```ruby
pod 'DyteiOSCore'
```

2. Install the client SDK from pod

```shell
pod install
```

3. Add the fonts and permission entries in `info.plist` - the strings are
   displayed in the permission pop-ups, feel free to customize them to suit your
   audience

```xml
<key>NSBluetoothPeripheralUsageDescription</key>
<string>We will use your Bluetooth to access your Bluetooth headphones.</string>
<key>NSBluetoothAlwaysUsageDescription</key>
<string>We will use your Bluetooth to access your Bluetooth headphones.</string>
<key>NSCameraUsageDescription</key>
<string>For people to see you during meetings, we need access to your camera.</string>
<key>NSMicrophoneUsageDescription</key>
<string>For people to hear you during meetings, we need access to your microphone.</string>
<key>NSPhotoLibraryUsageDescription</key>
<string>For people to share, we need access to your photos.</string>
<false/>
```

## Step 2: Initialize the SDK

The `DyteMobileClient` is the main class of the SDK. It is the main entry point
of the SDK. It is the only class that you need to instantiate in order to use
the SDK. To instantiate DyteMobileClient, you should use
`DyteiOSClientBuilder().build()`.

```swift
let meeting = DyteiOSClientBuilder().build()
```

Add listeners and implement callback stubs

```swift
meeting.addMeetingRoomEventsListener(meetingRoomEventsListener: self)
meeting.addParticipantEventsListener(participantEventsListener: self)
meeting.addSelfEventsListener(selfEventsListener: self)
```

## Step 3: Set the meeting properties

You can get the `roomName` and `authToken` using our backend APIs. Please refer
[this for room name](https://docs.dyte.io/api#/operations/createMeeting) &
[this for auth token](https://docs.dyte.io/api#/operations/addParticipant)

For V1 APIs:

```swift
let meetingInfo = DyteMeetingInfo(authToken: authToken,
                                          roomName: meetingText,
                                          enableAudio: true,
                                          enableVideo: true)

```

For V2 APIs: (Note: roomName is not required for v2 APIs)

```swift
let meetingInfo = DyteMeetingInfo(authToken: authToken,
                                          enableAudio: true,
                                          enableVideo: true)

```

## Step 4: Initialize

For V1 APIs:

```swift
meeting.doInit(dyteMeetingInfo: meetingInfo)
```

For V2 APIs:

```swift
dyteMobileClient?.doInit(dyteMeetingInfo_: meetingInfo)
```

## Step 5: Connect to the meeting

after `onMeetingInitCompleted()` Now to join in the room, use

```swift
meeting.joinRoom()
```

:::info Join room event listeners Once you call `joinRoom()`, you can listen to
callbacks for this action on meeting object by registering observer. :::

```swift
extension MeetingViewModel: DyteSelfEventsListener {
    func onMeetingRoomJoinStarted() {
        // meeting join started
    }

    func onMeetingRoomJoined() {
        // meeting room joined succesfully
    }

    func onMeetingRoomJoinFailed(exception: KotlinException) {
        // error in joining meeting room.
    }
  }
```

And voila, you just joined a room with Dyte's mobile SDK.

### Leave the room

Once the meeting is over, you can leave the meeting room.

To leave the meeting you can call `leaveRoom()` on `meeting` object.

```swift
meeting.leaveRoom()
```

:::info Leave room event listeners You can listen to `leaveRoom()` callbacks by
registering obsever on meeting object as follows :::

```swift
extension MeetingViewModel: DyteSelfEventsListener {
    func onMeetingRoomLeaveStarted() {
        // meeting room leave started
    }

    func onMeetingRoomLeft() {
        // meeting room leave completed
    }
}
```

### For iOS release builds

Disable `BITCODE` in your Podfile.

```dart
post_install do |installer|
   installer.pods_project.targets.each do |target|
       target.build_configurations.each do |config|
           config.build_settings['ENABLE_BITCODE'] = 'NO'
       end
   end
end
```
